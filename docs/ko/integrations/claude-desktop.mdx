**Claude Desktop** 통합을 통해 Claude 인터페이스에서 직접 MetaMCP 도구에 접근할 수 있습니다. Claude Desktop은 stdio 서버만 지원하므로, MetaMCP의 원격 엔드포인트에 연결하기 위해 로컬 프록시가 필요합니다.

## 사전 요구사항

시작하기 전에 다음이 필요합니다:

- **Claude Desktop** 설치 및 실행
- **MetaMCP** 로컬 실행 또는 배포됨
- MetaMCP에서 설정된 **활성 엔드포인트**
- **API 키** 생성 (인증이 활성화된 경우)

## 기본 설정

### mcp-proxy 사용 (권장)

MetaMCP 엔드포인트는 원격 전용 (SSE, Streamable HTTP)이므로, Claude Desktop이 연결하려면 로컬 프록시가 필요합니다. 테스트 결과, `mcp-proxy`가 API 키 인증에 권장되는 솔루션입니다.

<CodeGroup>
```json Streamable HTTP (권장)
{
  "mcpServers": {
    "MetaMCP": {
      "command": "uvx",
      "args": [
        "mcp-proxy",
        "--transport",
        "streamablehttp",
        "http://localhost:12008/metamcp/your-endpoint-name/mcp"
      ],
      "env": {
        "API_ACCESS_TOKEN": "sk_mt_your_api_key_here"
      }
    }
  }
}
```

```json SSE (대안)
{
  "mcpServers": {
    "MetaMCP": {
      "command": "uvx",
      "args": [
        "mcp-proxy",
        "http://localhost:12008/metamcp/your-endpoint-name/sse"
      ],
      "env": {
        "API_ACCESS_TOKEN": "sk_mt_your_api_key_here"
      }
    }
  }
}
```

```json 여러 엔드포인트
{
  "mcpServers": {
    "MetaMCP-Dev": {
      "command": "uvx",
      "args": [
        "mcp-proxy",
        "--transport",
        "streamablehttp",
        "http://localhost:12008/metamcp/dev-tools/mcp"
      ],
      "env": {
        "API_ACCESS_TOKEN": "sk_mt_dev_key"
      }
    },
    "MetaMCP-Research": {
      "command": "uvx",
      "args": [
        "mcp-proxy",
        "http://localhost:12008/metamcp/research-tools/sse"
      ],
      "env": {
        "API_ACCESS_TOKEN": "sk_mt_research_key"
      }
    }
  }
}
```
</CodeGroup>

## 설정 파일 위치

Claude Desktop의 설정 파일을 다음 위치에서 편집합니다:

- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\Claude\claude_desktop_config.json`
- **Linux**: `~/.config/claude/claude_desktop_config.json`

## 인증 방법

<AccordionGroup>
  <Accordion icon="key" title="API 키 인증">
    환경 변수를 사용하는 **가장 일반적인 방법**:

    ```json
    {
      "mcpServers": {
        "MetaMCP": {
          "command": "uvx",
          "args": [
            "mcp-proxy",
            "--transport",
            "streamablehttp",
            "http://localhost:12008/metamcp/your-endpoint-name/mcp"
          ],
          "env": {
            "API_ACCESS_TOKEN": "sk_mt_your_key_here"
          }
        }
      }
    }
    ```
  </Accordion>

  <Accordion icon="shield-minus" title="인증 없음">
    인증이 없는 **공개 엔드포인트**의 경우:

    ```json
    {
      "mcpServers": {
        "MetaMCP": {
          "command": "uvx",
          "args": [
            "mcp-proxy",
            "http://localhost:12008/metamcp/public-tools/sse"
          ]
        }
      }
    }
    ```
  </Accordion>
</AccordionGroup>

## 원격/운영 설정

원격 MetaMCP 인스턴스의 경우 localhost URL을 교체하세요:

```json
{
  "mcpServers": {
    "MetaMCP-Production": {
      "command": "uvx",
      "args": [
        "mcp-proxy",
        "--transport",
        "streamablehttp",
        "https://your-metamcp-domain.com/metamcp/your-endpoint-name/mcp"
      ],
      "env": {
        "API_ACCESS_TOKEN": "sk_mt_production_key"
      }
    }
  }
}
```

## 중요 사항

- `your-endpoint-name`을 실제 엔드포인트 이름으로 **교체**하세요
- `sk_mt_your_api_key_here`를 MetaMCP API 키로 **교체**하세요
- **mcp-proxy**는 stdio와 HTTP/SSE 간의 프로토콜 변환을 처리합니다
- **환경 변수**는 API 키를 전달하는 안전한 방법입니다
- 자세한 문제 해결은 [이슈 #76](https://github.com/metatool-ai/metamcp/issues/76)을 참조하세요
