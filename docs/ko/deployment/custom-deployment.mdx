이 가이드는 예시로 Ubuntu를 실행하는 DigitalOcean VPS에 MetaMCP를 처음부터 배포하는 방법을 안내합니다.

## 사전 요구사항

- DigitalOcean 계정
- VPS를 가리키는 도메인 이름
- Linux 커맨드 라인 기본 지식

## 시스템 요구사항

MetaMCP는 최적 성능을 위해 최소 **2GB-4GB 메모리**가 필요합니다. MCP 서버 사전 할당 및 Docker 작업으로 인해 인스턴스가 클수록 성능이 더 좋습니다.

**권장 DigitalOcean Droplet:**
- **기본/일반**: 2GB RAM, 1 vCPU, 50GB SSD (월 $12)
- **더 나은 성능**: 4GB RAM, 2 vCPU, 80GB SSD (월 $24)

## 1단계: VPS 생성 및 설정

### 1.1 Droplet 생성

1. DigitalOcean 계정에 로그인합니다
2. "생성" → "Droplets" 클릭
3. OS로 **Ubuntu 22.04 LTS** 선택
4. 최소 2GB RAM이 있는 플랜 선택
5. 사용자와 가까운 데이터센터 지역 선택
6. 보안 접근을 위해 SSH 키 추가
7. Droplet 생성

### 1.2 초기 서버 설정

SSH를 통해 서버에 연결합니다:

```bash
ssh root@your_server_ip
```

시스템 업데이트:

```bash
apt update && apt upgrade -y
```

필수 패키지 설치:

```bash
apt install -y curl wget git ufw nginx certbot python3-certbot-nginx
```

방화벽 설정:

```bash
ufw allow OpenSSH
ufw allow 'Nginx Full'
ufw enable
```

비루트 사용자 생성 (선택사항이지만 권장):

```bash
adduser metamcp
usermod -aG sudo metamcp
# 새 사용자로 전환
su - metamcp
```

## 2단계: Docker 및 Docker Compose 설치

### 2.1 Docker 설치

패키지 인덱스를 업데이트하고 사전 요구사항을 설치합니다:

```bash
sudo apt update
sudo apt install -y apt-transport-https ca-certificates curl gnupg lsb-release
```

Docker의 공식 GPG 키와 저장소를 추가합니다:

```bash
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
sudo add-apt-repository "deb [arch=amd64] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable"
```

Docker CE 설치:

```bash
sudo apt update
sudo apt install -y docker-ce
```

Docker가 실행 중인지 확인:

```bash
sudo systemctl status docker
```

### 2.2 Docker Compose 설치

Docker Compose를 다운로드하고 설치합니다:

```bash
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

### 2.3 Docker 사용자 설정 (선택사항)

`sudo` 없이 Docker 명령어를 실행하려면:

```bash
sudo usermod -aG docker $USER
```

그룹 변경사항을 적용하려면 로그아웃했다가 다시 로그인하거나 다음을 실행합니다:

```bash
newgrp docker
```

## 3단계: MetaMCP 배포

### 3.1 저장소 클론

```bash
cd /opt
sudo git clone https://github.com/metatool-ai/metamcp.git
sudo chown -R $USER:$USER metamcp
cd metamcp
```

### 3.2 환경 설정

```bash
cp example.env .env
```

도메인과 설정으로 `.env` 파일을 편집합니다:

```bash
nano .env
```

**⚠️ 중요 보안 주의사항**: 운영 환경이므로 `POSTGRES_PASSWORD`와 `BETTER_AUTH_SECRET`을 기본값에서 반드시 변경하세요. 또한 HTTPS를 사용해야 합니다. 보안 시크릿을 생성하는 일반적인 방법은 다음과 같습니다:

```bash
openssl rand -hex 32 | base64
```

업데이트해야 할 주요 설정:

```env
# 도메인 URL (운영 환경에서는 반드시 HTTPS 사용!)
APP_URL=https://yourdomain.com

# 데이터베이스 - 기본값에서 비밀번호를 변경하세요!
DATABASE_URL=postgresql://postgres:YOUR_SECURE_PASSWORD@db:5432/metamcp
POSTGRES_PASSWORD=YOUR_SECURE_PASSWORD

# 보안 시크릿 생성 - 예시 값을 사용하지 마세요!
BETTER_AUTH_SECRET=your-super-secret-key-here
ENCRYPTION_KEY=your-32-character-encryption-key

# 선택사항: 필요한 경우 OIDC 설정
# OIDC_CLIENT_ID=your-oidc-client-id
# OIDC_CLIENT_SECRET=your-oidc-client-secret
# OIDC_DISCOVERY_URL=https://your-provider.com/.well-known/openid-configuration
```

### 3.3 운영용 Docker Compose 업데이트

충돌을 피하기 위해 볼륨 이름이 올바른지 `docker-compose.yml`을 편집합니다:

```bash
nano docker-compose.yml
```

충돌을 피하기 위해 볼륨 섹션을 업데이트합니다:

```yaml
volumes:
  metamcp_postgres_data:
    driver: local
```

### 3.4 MetaMCP 시작

```bash
# 이미지를 가져오고 서비스 시작
docker-compose up -d

# 서비스가 실행 중인지 확인
docker-compose ps

# 필요한 경우 로그 보기
docker-compose logs -f
```

## 4단계: Nginx 리버스 프록시 설정

### 4.1 Nginx 설정 생성

새 사이트 설정을 생성합니다:

```bash
sudo nano /etc/nginx/sites-available/metamcp
```

다음 설정을 추가합니다 (`yourdomain.com`을 실제 도메인으로 교체):

```nginx
server {
    listen 80;
    server_name yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name yourdomain.com;

    # SSL 설정 (Certbot이 관리)
    ssl_certificate /etc/letsencrypt/live/yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/yourdomain.com/privkey.pem;
    include /etc/letsencrypt/options-ssl-nginx.conf;
    ssl_dhparam /etc/letsencrypt/ssl-dhparams.pem;

    # 보안 헤더
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    add_header Referrer-Policy "no-referrer-when-downgrade" always;

    location / {
        proxy_pass http://localhost:12008;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # MCP 연결을 위한 SSE 특화 최적화
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 86400s;  # 장시간 SSE 연결을 위한 24시간
        proxy_send_timeout 86400s;

        # SSE를 위한 적절한 연결 처리가 포함된 HTTP/1.1
        proxy_set_header Connection '';
        proxy_http_version 1.1;

        # 더 나은 SSE 지원을 위한 추가 헤더
        proxy_set_header Cache-Control 'no-cache';
        proxy_set_header X-Accel-Buffering 'no';
    }

    # 선택사항: 파일 업로드를 위한 클라이언트 최대 본문 크기 증가
    client_max_body_size 100M;
}
```

### 4.2 사이트 활성화

```bash
# 사이트 활성화
sudo ln -s /etc/nginx/sites-available/metamcp /etc/nginx/sites-enabled/

# 기본 사이트 제거
sudo rm /etc/nginx/sites-enabled/default

# nginx 설정 테스트
sudo nginx -t

# nginx 시작
sudo systemctl enable nginx
sudo systemctl start nginx
```

## 5단계: Let's Encrypt SSL 인증서

### 5.1 SSL 인증서 획득

초기 인증서를 위해 임시로 HTTP 전용 설정을 사용합니다:

```bash
# 임시 HTTP 전용 설정 생성
sudo nano /etc/nginx/sites-available/metamcp-temp
```

이 임시 설정을 추가합니다:

```nginx
server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:12008;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

임시 설정 활성화:

```bash
sudo ln -sf /etc/nginx/sites-available/metamcp-temp /etc/nginx/sites-enabled/metamcp
sudo nginx -t && sudo systemctl reload nginx
```

인증서 획득:

```bash
sudo certbot --nginx -d yourdomain.com
```

### 5.2 전체 설정 복원

인증서 획득 후 전체 설정을 복원합니다:

```bash
sudo ln -sf /etc/nginx/sites-available/metamcp /etc/nginx/sites-enabled/metamcp
sudo nginx -t && sudo systemctl reload nginx
```

### 5.3 자동 갱신 설정

```bash
# 자동 갱신 테스트
sudo certbot renew --dry-run

# 자동 갱신을 위해 crontab에 추가
sudo crontab -e
```

하루에 두 번 갱신을 확인하는 이 줄을 추가합니다:

```
0 12 * * * /usr/bin/certbot renew --quiet
```
