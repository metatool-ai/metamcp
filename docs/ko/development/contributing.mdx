MetaMCP에 대한 기여를 환영합니다! 이 종합 가이드는 버그 수정, 기능 추가, 문서 개선 등 프로젝트에 기여를 시작하는 데 도움이 됩니다.

## 시작하기

### 사전 요구사항

기여하기 전에 다음이 필요합니다:

- **Node.js 18+** 및 **pnpm** 설치
- PostgreSQL 실행 및 테스트를 위한 **Docker**
- 버전 관리를 위한 **Git**
- TypeScript, React, MCP 프로토콜에 대한 **기본 이해**

### 개발 환경 설정

<AccordionGroup>
  <Accordion icon="github" title="Fork 및 Clone">
    1. GitHub에서 **저장소를 Fork**합니다
    2. Fork를 로컬에 **Clone**합니다:
       ```bash
       git clone https://github.com/YOUR_USERNAME/metamcp.git
       cd metamcp
       ```
    3. **upstream 원격 추가**:
       ```bash
       git remote add upstream https://github.com/metatool-ai/metamcp.git
       ```
  </Accordion>

  <Accordion icon="package" title="의존성 설치">
    pnpm을 사용하여 프로젝트 의존성을 설치합니다:

    ```bash
    pnpm install
    ```

    이것은 모노레포의 모든 워크스페이스에 대한 의존성을 설치합니다.
  </Accordion>

  <Accordion icon="cog" title="환경 설정">
    개발 환경을 설정합니다:

    ```bash
    cp example.env .env
    ```

    개발 설정에 맞게 `.env` 파일을 수정합니다.
  </Accordion>

  <Accordion icon="database" title="데이터베이스 설정">
    Docker를 사용하여 PostgreSQL을 시작합니다:

    ```bash
    docker compose up -d postgres
    # 또는 전체 스택 시작
    docker compose up -d
    ```

    최초 마이그레이션 (먼저 `.env.local` 편집)
    ```bash
    cd apps/backend

    pnpm db:migrate:dev
    ```
  </Accordion>
</AccordionGroup>

### 변경사항 테스트

<AccordionGroup>
  <Accordion icon="flask-conical" title="로컬 테스트">

  </Accordion>

  <Accordion icon="docker" title="Docker 테스트">
    운영 호환성을 위해 Docker로 테스트합니다:

    - MetaMCP 버전: [예: 1.0.0]

    ## 추가 컨텍스트

## 전문 기여

### OIDC 공급자 설정

MetaMCP는 엔터프라이즈 SSO를 위한 OpenID Connect를 지원합니다. OIDC 기능 작업 시:

<AccordionGroup>
  <Accordion icon="shield" title="OIDC 설정">
    **필요한 환경 변수:**

    ```bash
    # 필수
    OIDC_CLIENT_ID=your-oidc-client-id
    OIDC_CLIENT_SECRET=your-oidc-client-secret
    OIDC_DISCOVERY_URL=https://your-provider.com/.well-known/openid-configuration
    OIDC_AUTHORIZATION_URL=https://your-provider.com/auth/authorize

    # 선택사항
    OIDC_PROVIDER_ID=oidc
    OIDC_SCOPES=openid email profile
    OIDC_PKCE=true
    ```
  </Accordion>

  <Accordion icon="flask-conical" title="OIDC 테스트">
    **OIDC 개발을 위해:**

    1. 테스트 공급자 사용 (Auth0, Keycloak)
    2. 리디렉션 URI 설정: `${APP_URL}/api/auth/oauth2/callback/oidc`
    3. 인증 흐름 테스트
    4. 데이터베이스에서 사용자 생성 확인
    5. 문제 해결을 위한 디버그 로깅 활성화
  </Accordion>
</AccordionGroup>

### 데이터베이스 변경

데이터베이스 스키마를 변경할 때:

<AccordionGroup>
  <Accordion icon="database" title="스키마 마이그레이션">
    **마이그레이션 생성:**

    ```bash
    # 스키마 변경 후 마이그레이션 생성
    cd apps/backend
    pnpm db:generate

    # 마이그레이션 적용
    pnpm db:migrate:dev # PG 관련 환경 변수에 env.local 사용

    # 데이터베이스 초기화 (개발 전용)
    pnpm db:reset
    ```
  </Accordion>

  <Accordion icon="table" title="새 테이블 추가">
    **데이터베이스 개발 워크플로우:**

    1. `apps/backend/src/db/schema.ts`에서 스키마 업데이트
    2. `apps/backend/src/db/repositories/`에 저장소 생성
    3. `apps/backend/src/db/serializers/`에 직렬화기 생성
    4. `apps/backend/src/trpc/`에 tRPC 프로시저 추가
    5. `packages/zod-types/`에서 프론트엔드 타입 업데이트
    6. 마이그레이션 생성 및 적용
  </Accordion>
</AccordionGroup>

### 프론트엔드 개발

<AccordionGroup>

    - 기존 디자인 패턴 따르기

  </Accordion>

  <Accordion icon="globe" title="국제화">

    2. 다른 로케일 업데이트 또는 번역 표시
    3. `useTranslations()` 훅 사용
    4. 다른 언어로 테스트
    5. 텍스트 확장이 레이아웃을 깨뜨리지 않는지 확인
